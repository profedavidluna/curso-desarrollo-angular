# MODULO 2: EJERCICIOS DE PRÁCTICA

## Ejercicio 7: Sistema de Carrito de Compras (intermedio)

### Requerimientos:
- Crear un sistema de carrito de compras donde los usuarios puedan agregar y quitar productos.
- Cada producto debe tener un nombre, precio y cantidad.
- El carrito debe mostrar el total a pagar.

### Arquitectura técnica:
- Frontend: Angular
- Backend: Node.js con Express
- Base de datos: MongoDB para almacenar productos y pedidos.

### Criterios de evaluación:
- Funcionalidad completa sin errores
- Interfaz de usuario atractiva y responsiva
- Código bien estructurado y documentado.

### Instrucciones de entrega:
- Subir el código a un repositorio en GitHub
- Incluir un archivo README.md con instrucciones de uso.

## Ejercicio 8: Simulación de Chat en Tiempo Real (intermedio-avanzado)

### Requerimientos:
- Crear una aplicación de chat donde múltiples usuarios puedan ingresar y enviar mensajes en tiempo real.
- Almacenar el historial de chat en una base de datos.

### Arquitectura técnica:
- Frontend: Angular
- Backend: Node.js con Socket.IO para la comunicación en tiempo real.
- Base de datos: Firebase para el almacenamiento de mensajes.

### Criterios de evaluación:
- Funcionamiento fluido del chat en tiempo real.
- Diseño funcional y atractivo.
- Código claro y modular.

### Instrucciones de entrega:
- Publicar el código en GitHub.
- Proveer documentación clara sobre cómo ejecutar la aplicación.

## Ejercicio 9: Sistema Completo de Gestión de Usuarios CRUD (avanzado)

### Requerimientos:
- Desarrollar un sistema que permita crear, leer, actualizar y eliminar usuarios.
- Implementar autenticación para que los usuarios puedan registrarse y loguearse.

### Arquitectura técnica:
- Frontend: Angular con Reactive Forms
- Backend: Node.js con Express
- Base de datos: PostgreSQL para almacenar datos de usuarios.

### Criterios de evaluación:
- Todas las operaciones CRUD implementadas con éxito.
- Manejo de errores adecuado y validaciones en el frontend.
- Uso de buenas prácticas de programación.

### Instrucciones de entrega:
- El código debe ser compartido a través de un repositorio en GitHub.
- Incluir un archivo de documentación con detalles sobre cómo usar el sistema.
